Float, quem nunca usou e teve problema com esta propriedade? Problemas com float é algo comum para quem desenvolve layouts com mais de uma coluna.

O problema basicamente é quando temos em nosso layout alguns elementos flutuantes que apesar de bem posicionados acabam saindo fora do fluxo normal do documento e tendo um comportamento totalmente diferente do que esperamos e como resultado, temos um layout totalmente bagunçado. Mas,não se despere, tenho uma resposta e uma solução para você.

Infelizmente este problema acontece porque o pai do elemento que tem a propriedade float, não consegue calcular sua altura automaticamente e com isso ajustar o elemento flutuante em seu fluxo normal.

Existem várias formas de resolver esse problema, mas a forma mais simples é usar um class chamado .clearfix, veja só:

{% highlight ruby linenos %} .clearfix:after{ content: ""; display: table; clear: both; } {% endhighlight %}

Nesse código usamos o pseudo-class :after, isso significa que queremos que algo aconteça depois do elemento que tenha o class .clearfix.

No class .clearfix temos alguns atributos que fazem toda diferença veja só:

  • O uso do content: ""; faz com que seja criado um conteúdo vazio após o elemento. Lembrando que dentro desse atributo pode conter textos.
  • Temos também um atributo display: table; que cria célula de uma tabela anônima sem precisar de todas as tags que normalmente uma tabela usa.
  • Os atributos content: ""; e display: table; juntos, fazem com que se crie um novo bloco no fluxo do documento, impedindo o colapso das margens.
  • E por ultimo e muito importante clear:both;, este atributo provoca uma quebra no float fazendo com que o elemento não sofra influência alguma de nenhum objeto com float:left; ou float:right;.

E se sua preocupação é se este método é suportado pelos navegadores, fique tranquilo pois é totalmentez suportado do IE 8+.

Resumindo, o que você provavelmente deve fazer é criar um class em seu projeto chamada .clearfix (ou o nome de sua preferência) e aplicar aos seus elementos. Esta é a maneira mais simples e mais limpa para lidar com seus elementos com float.
